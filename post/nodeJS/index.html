node基本介绍

Node.js 是一个基于 Chrome V8 引擎的 JavaScript 运行环境。
Node.js 使用了一个事件驱动、非阻塞式 I/O 的模型，使其轻量又高效。

  • node.js不是一门语言

  • node.js不是库、不是框架

  • node.js是一个javascript运行环境

  • node.js可以解析和执行javascript代码

  • 以前只有浏览器可以执行JavaScript代码

  • 没有bom和dom

  • node为这个javascript提供了一些服务器级别的api，
    例如文件读写
    网络服务的构建
    网络通信
    http服务器
    等处理

  • 代码只是具有特定格式的字符串

  • 引擎可以认识它，并帮你去解析和执行

  • Google Chrome的V8引擎是目前公认的解析javascript代码最快的

  • nodeJS的作者把Google Chrome中的V8引擎移植出来，开发了一个独立的javascript运行环境

  • npm是世界上最大的开源生态系统

  • 绝大多数的javascript相关的包都放在了npm上，开发人员使用下载非常方便

node能做什么

  • web服务器后台
  • 命令行工具 如npm hexo（构建github博客）

node包加载机制

  1. 优先从缓存加载
  2. 当前目录的node_moudels目录中查找对应的包名
    然后找到后找其中的package.json文件，获取其中的mian对应的文件索引
    然后加载这个文件
    如果找不到这个文件或者找不着对应的路径则默认加载这个包文件下的index.js
  3. 如果都没找到会去上一级目录的node_moudels文件中找，知道磁盘根路径
  4. 路径形式的模块
    ./ 当前目录，不可省略
    ../ 上一级目录，不可省略

node路径中的/

  • 读取文件中可以省略./，和加./一样都是去当前目录中找
  • 引入包是不能省略./，不然node会当做核心模块或者第三方模块来加载，找不到后报错
  • 读取文件省略.只用/会去当前文件所在的磁盘根目录中找，找不到也会报错

express

const express=require('express')
const bodyParser=require('body-parser')

const app=express()
// 开发静态资源
app.use('/public/',express.static('./public/'))
app.use('/static/',express.static('./node_modules'))
//配置模板引擎
app.engine('html', require('express-art-template'));
//修改模板文件路径
//app.set('views','./htmls/')

// 配置body-parser中间件
app.use(bodyParser.urlencoded({ extended: false }))
// 将post参数转化为json
app.use(bodyParser.json())

app.get('/', function (req, res) {
    res.render('index.html', {
        user: {
            name: 'aui',
            tags: ['art', 'template', 'nodejs']
        }
    });
});
//重定向
app.get('/redirect',function(req,res){
    res.redirect('/')
})

app.listen(3000,()=>console.log('service running at 127.0.0.1:3000'))
